Data Structures & Algorithms

Target: 150–200 problems for product companies; 80–100 for mass IT recruiters (mostly Easy + selected Medium).

Topic order

  1. Arrays, strings, hashing
  2. Two pointers, sliding window
  3. Stacks, queues, linked lists
  4. Trees, BST, traversals
  5. Graphs (BFS, DFS)
  6. Dynamic programming (last)

How to solve a coding interview question

  1. Clarify inputs, outputs, edge cases, and constraints.
  2. Describe brute force and its time/space complexity.
  3. Optimize (pattern recognition) and get interviewer buy-in.
  4. Code cleanly; use meaningful variable names.
  5. Walk through a sample test case.
  6. State final complexity and trade-offs.

Project interview prep

Be ready to answer in 5 minutes:

  • What problem does it solve?
  • Tech stack and why you chose it
  • Database schema or API design
  • Hardest bug and how you fixed it
  • What you would improve with more time
